# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**RB Drum Sampler** is a VST3 JUCE plugin that receives Rock Band (RB) MIDI drum input and plays back real drum samples. It is built for use with Rock Band clone/emulator setups where the game outputs MIDI on a known note-number scheme.

## Build

Prerequisites: CMake ≥ 3.22, Visual Studio (MSVC), Git (CPM fetches JUCE automatically).

```powershell
# Configure (downloads JUCE 7.0.12 via CPM on first run — takes a few minutes)
cmake -B build -S .

# Build the VST3
cmake --build build --config Release

# Or Debug
cmake --build build --config Debug
```

The compiled VST3 lands in `build/RBDrumSampler_artefacts/`. No install step is wired up; copy the `.vst3` bundle manually to your DAW's plugin folder.

There are no automated tests.

## Audio Samples

The plugin embeds OGG files at build time via `juce_add_binary_data`. All nine samples must exist in `audio/` before configuring CMake:

| File | Triggered by |
|------|-------------|
| `Kick1.ogg` | MIDI 96 |
| `Snare1.ogg` | MIDI 97 |
| `HhClosed.ogg` | MIDI 98 (hi-hat default) |
| `Ride1.ogg` | MIDI 99 (blue cymbal default) |
| `Crsh1.ogg` | MIDI 100 (green cymbal default) |
| `Crsh2.ogg` | MIDI 101 (Y+G combo crash) |
| `HiMidTom.ogg` | MIDI 110 (yellow tom override) |
| `LoMidTom.ogg` | MIDI 111 (blue tom override) |
| `LowFlrTom.ogg` | MIDI 112 (green tom override) |

## Architecture

### MIDI Remapping (`src/PluginProcessor.cpp`)

The core logic is in `processBlock`. Rock Band sends drums on a fixed note scheme; the processor remaps those to the internal sample trigger notes in two passes:

**Pass 1** — scan the block to detect which RB notes are present (needed for combo detection).

**Pass 2** — remap each note-on:

| RB note | Normal target | Condition → alternate target |
|---------|---------------|------------------------------|
| 96 | 96 (Kick) | — |
| 97 | 97 (Snare) | — |
| 98 | 98 (Hi-hat) | marker 110 present → 110 (Yellow Tom); Y+G combo → 101 (Crash 2) |
| 99 | 99 (Ride) | marker 111 present → 111 (Blue Tom) |
| 100 | 100 (Crash 1) | marker 112 present → 112 (Green Tom) |
| 110–112 | dropped | tom markers carry no audio; they only inform the remap above |

**Y+G combo**: both notes 98 and 100 arrive in the same block **without** tom markers 110 or 112 — this signals a double-crash hit and routes 98 → Crash 2 (101) instead of hi-hat.

**Note-offs are intentionally dropped** for all RB drum pitches (96–112). RB PART DRUMS note-offs arrive almost immediately after note-ons (~0.05 s), which would choke the sample. Voices are left to decay naturally.

### Sample Loading (`loadSamples`)

Called from `prepareToPlay`. Each OGG file is pulled from the embedded `BinaryData` namespace (generated by `juce_add_binary_data`), decoded, and registered as a `juce::SamplerSound` on a single MIDI note. Attack is 0, release is 0.1 s, max length 10 s.

### Voice Pool

16 `juce::SamplerVoice` instances are created in the constructor and shared across all sounds. Voice stealing follows JUCE's default policy (oldest voice).

### Editor (`src/PluginEditor.cpp`)

Full UI with per-drum controls for all nine channels (Kick, Snare, Hi-Hat, Ride, Crash 1, Crash 2, Hi Tom, Mid Tom, Floor Tom). Each channel has three sliders:

- **Volume** (0.0–2.0, default 1.0)
- **Attack** (0–500 ms, default ~1 ms)
- **Release** (10 ms–3 s, default varies by drum)

Two global buttons: **Mute All** (toggles silence without touching sliders; turns red when active) and **Reset to Default** (restores all sliders to factory values).

### Parameter Persistence

All slider values are backed by `AudioProcessorValueTreeState` (APVTS) and serialized to XML via `getStateInformation` / `setStateInformation` — they persist with the DAW project. The Mute All button state is intentionally excluded from persistence and resets to off on every load.

### MIDI — Expert difficulty only

Notes 96–112 correspond exclusively to the Expert difficulty lane of `PART DRUMS`. Notes from other lanes (Easy: 60–64, Medium: 72–76, Hard: 84–88) are not mapped and are silently ignored.
